🚀 FAST DEPLOYMENT PATH (2 MINUTES):
───────────────────────────────────────────────────────────────────────────────

1. CREATE DIRECTORY & COPY FILES
   mkdir pailit && cd pailit
   
   Create folders:
   - mkdir public
   - mkdir src
   
   Copy files with correct names:
   - pailit-package.json          → package.json
   - pailit-App.jsx               → src/App.jsx
   - pailit-App.css               → src/App.css
   - pailit-index.js              → src/index.js
   - pailit-index.html            → public/index.html
   - pailit-.gitignore            → .gitignore

2. INITIALIZE GIT REPO
   git init
   git add .
   git commit -m "Initial commit: Pailit MVP"

3. CREATE GITHUB REPO
   Go to: https://github.com/new
   Name: pailit
   (Don't init with README, .gitignore, license)
   
   Then:
   git remote add origin https://github.com/YOUR_USERNAME/pailit.git
   git branch -M main
   git push -u origin main

4. DEPLOY TO VERCEL (FREE)
   Go to: https://vercel.com/dashboard
   Click: "Add New" → "Project"
   Select: pailit repo from GitHub
   Click: "Deploy"
   
   ✅ DONE! App live in 30 seconds.
   Your URL: https://pailit-[random].vercel.app

❓ COMMON QUESTIONS:
───────────────────────────────────────────────────────────────────────────────

Q: Do I need to pay for Vercel?
A: No, free tier is perfect for this. No credit card needed.

Q: Can I use my own domain?
A: Yes, Vercel allows free custom domain setup in project settings.

Q: Will notifications work on mobile?
A: Yes, as long as you grant browser permission. Some restrictions on iOS Safari.

Q: Can multiple people use the same deployment?
A: Yes! Each person's browser keeps their own local data (localStorage).

Q: How do I update the app after deploying?
A: Make changes locally, git push to GitHub, Vercel auto-deploys.

Q: Can I customize the sarcasm messages?
A: Yes! Edit getSarcasm() function in src/App.jsx and push changes.
